# PPT Creator - Intake Form

This document contains the minimal 10-item questionnaire used to gather context when creating presentations.

## Minimal 10 Questions

When key information is missing, ask these questions. If the user doesn't respond after 2 prompts, use the safe defaults listed below.

### 1. Who is the audience?
**Default**: General public
**Purpose**: Determines tone, complexity, and terminology

### 2. What is the core objective?
**Default**: "Understand and accept" a proposition
**Purpose**: Defines the main message and goal

### 3. What is the desired action/decision?
**Default**: Agree to move to next step after the meeting
**Purpose**: Establishes clear call-to-action

### 4. Duration & slide count limit?
**Default**: 15-20 minutes, 12-15 slides
**Purpose**: Determines depth and pacing

### 5. What tone & style?
**Default**: Professional, clear, friendly
**Purpose**: Sets the voice and presentation style

### 6. Topic scope & boundaries?
**Default**: Given topic + 1 layer of related context
**Purpose**: Prevents scope creep and maintains focus

### 7. Must-include points or taboos?
**Default**: None
**Purpose**: Ensures critical content is included and sensitive topics avoided

### 8. Available data/tables?
**Default**: None; can generate structure placeholder + list required fields
**Purpose**: Determines if real data visualizations can be created

### 9. Brand & visual constraints?
**Default**: Built-in neutral theme
**Purpose**: Ensures visual consistency with brand guidelines

### 10. Deliverable format preference?
**Default**: slides.md + optional PNG charts; export PPTX if available
**Purpose**: Determines output format and tools needed

## Usage Notes

- Ask questions conversationally, not as a rigid form
- Group related questions together
- If user provides partial information, fill in gaps with defaults
- Always document which defaults were used in speaker notes
- Prioritize questions 1-4 as most critical for structure
